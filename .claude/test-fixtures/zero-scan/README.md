# Zero-Scan Fixture

Tests that `evdb-dev-v2` can implement a slice without reading any existing blueprint code.

## What it tests

**Property**: The AI fills in business logic using only `TODO_CONTEXT.md` and the scaffolded
files for the current slice. It never reads other slices, stream factories, adapters, or
endpoint files to discover patterns.

## Slice: Record Deposit

- **Command**: `RecordDeposit(accountId, amount, currency, recordedAt [generated])`
- **Events**: `DepositRecorded`, `DepositRejected(reason [generated])`
- **Spec 1** — "amount is zero or negative" → `DepositRejected`
- **Spec 2** — "amount is positive" → `DepositRecorded`

The business logic is intentionally obvious so the AI has no reason to scan existing code.
Any violations are structural — the AI was looking for a pattern it should already know.

## How to run

```bash
# 1. Start a scan session
python3 .claude/skills/evdb-dev-v2/scripts/scan_session.py clear
python3 .claude/skills/evdb-dev-v2/scripts/scan_session.py start \
  --slice recorddeposit --context Demo --root .

# 2. Copy the fixture eventmodel into place (in a worktree or temp dir)
#    OR point the scaffold at the fixture's eventmodel directly:
python3 .claude/skills/evdb-scaffold/scripts/evdb_scaffold.py \
  --root . \
  --slice recorddeposit \
  --eventmodel-root .claude/test-fixtures/zero-scan/eventmodel

# 3. Invoke evdb-dev-v2 Step 3 (fill in TODOs)
#    The scan guard will log every Read call automatically via the hook.

# 4. Assert zero violations
python3 .claude/skills/evdb-dev-v2/scripts/scan_session.py assert

# 5. See the report (whether pass or fail)
python3 .claude/skills/evdb-dev-v2/scripts/scan_session.py report

# 6. Stop session
python3 .claude/skills/evdb-dev-v2/scripts/scan_session.py stop
```

## Learning loop

If `assert` exits with code 1, run `report` to see which files were scanned.
For each violation:

1. Identify what pattern the AI was looking for
2. Check if `TODO_CONTEXT.md` (generated by scaffold) already explains it
3. If not → update the `_gen_todo_context()` function in `evdb_scaffold.py`
   to include that pattern inline
4. Re-run from step 1 — goal is zero violations

Over time `TODO_CONTEXT.md` becomes the complete reference and no scanning is needed.
